# CarRent - 汽车租赁管理系统

## 项目简介

CarRent 是一个基于 C++ 开发的命令行汽车租赁管理系统，旨在实现车辆信息管理、租车业务、退车业务、统计报表等信息化管理。系统采用动态链表与文件技术相结合，提供高效、稳定的数据操作能力。

- **包名**: top.rayawa.car_rent
- **开发语言**: C++11
- **开发环境**: 跨平台（Windows / macOS / Linux）
- **构建工具**: Make / CMake

## 项目结构

```
│   ├── menu2_renter.cpp   # 租车用户信息管理
│   ├── menu3_rent.cpp     # 租车业务办理
│   ├── menu4_return.cpp   # 退车业务办理

│   ├── menu6_statistics.cpp # 统计汇总

├── CMakeLists.txt         # CMake 构建配置

└── README.md              # 本文件
```

## 功能模块

| 序号 | 功能 | 分数 | 说明 |
|------|------|------|------|
| 1 | 车辆信息管理 | 5 | 增删改查车辆信息（品牌、型号、颜色、车牌号、日租金等） |
| 2 | 租车用户信息管理 | 5 | 增删改查用户信息（姓名、驾照号、身份证号、驾龄等） |
| 3 | 租车业务办理 | 10 | 选择可租车辆，录入租车日期和预计归还日期，自动计算押金 |
| 4 | 退车业务办理 | 10 | 退车处理，自动计算费用和退款 |
| 5 | 票据生成 | 10 | 租车/退车自动生成格式化的中文票据 |
| 6 | 输出全部信息 | 10 | 列表展示所有车辆、用户、租车记录 |
| 7 | 信息查询 | 10 | 多条件查询（按品牌、类型、颜色、状态、姓名、驾照号等） |
| 8 | 信息统计汇总 | 5 | 按车型/品牌/颜色/日期等多维度统计 |
| 9 | 退出系统 | 5 | 安全退出 |
| 10 | 设置/修改密码 | 5 | 首次设置密码，支持修改密码，密码长度 6~16 位 |

### 代码规范（10 分）

- 标识符采用驼峰命名法，函数名采用动词+名词结构
- 每个函数单一职责，长度控制在 80 行以内
- 严格控制全局变量使用，数据通过参数传递
- 关键业务逻辑均添加注释
- 文件头部块注释、函数文档注释完整

### 链表应用（10 分）

- 使用通用双向链表模板类 `LinkedList<T>` 实现车辆、用户、租车记录的全部数据管理
- 链表与文件读写相结合，实现数据持久化

## 构建与运行

### 方式一：使用 Make

```bash
# 编译
make

# 运行
./CarRent

# 清理
make clean

# 清理（包括数据文件）
make distclean
```

### 方式二：使用 CMake

```bash
# 创建构建目录
mkdir build && cd build

# 配置
cmake ..

# 编译
make

# 运行
./CarRent
```

### 方式三：使用 g++ 直接编译

```bash
g++ -std=c++11 -o CarRent src/main.cpp
./CarRent
```

## 使用说明

1. **首次运行**：系统提示设置登录密码（6~16 位）
2. **登录**：输入正确的密码进入系统
3. **主菜单**：选择 1~9 进入各功能模块
4. **车辆管理**：先添加车辆信息，再办理租车业务
5. **用户管理**：添加租车用户信息
6. **租车**：选择可用车辆和用户，系统自动计算押金并生成票据
7. **退车**：输入租车编号，系统自动计算费用并生成退车收据
8. **查询统计**：多维度查询和统计分析
9. **扩展功能**：数据可视化图表、报表导出、操作日志查看

## 开发说明

- 程序入口：`src/main.cpp`
- 所有业务逻辑通过 Manager 类管理，UI 类负责交互
- 数据通过二进制文件持久化，链表在内存中操作
- 密码使用 XOR 简单加密存储
- 所有增删改操作自动记录到 `log.txt`
